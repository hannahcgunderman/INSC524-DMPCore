Specifications for DMPHub Element Set

Contains specification tables for 18 base elements and 3 wrapper elements

#1
Name fundAgency
Label Agency
URI http://www.utkmetadata.org/specification/utk_namespace/fundAgency
Definition The funding agency for which the DMP is being prepared.
Obligation & Occurrence Required
Example <fundAgency termsource="LCNAF" termsourceid="n2006182431">National Science Foundation (U.S.). Subcommittee on Nanoscale Science, Engineering, and Technology</fundAgency>
Comments Has LCNAF attributes and local vocabularies available

#2
Name template
Label Template
URI http://www.utkmetadata.org/specification/utk_namespace/template
Definition Templates used to create the DMP.
Obligation & Occurrence Optional, Repeatable
Example <template>DMP Tool</template>
Comments Has local vocabulary available

#3
Name creator
Label Creator
URI http://purl.org/dc/terms/creator
Definition The creator of the DMP.
Obligation & Occurrence Required, Repeatable
Example <creator>Avenal, Richmond</creator>
Comments Has local vocabulary available

#4
Name PI
Label PI
URI http://www.utkmetadata.org/specification/utk_namespace/PI
Definition The PI(s) of the research project being described in the DMP.
Obligation & Occurrence Required, Repeatable
Example <PI>Barber, Jen</PI>
Comments Has local vocabulary available

#5
Name contributor
Label Contributor
URI http://purl.org/dc/terms/contributor
Definition Any other individuals who contributed to the research project described in the DMP (may be different from the person who prepared the DMP).
Obligation & Occurrence Optional, Repeatable
Example <contributor>Moss, Maurice</contributor>
Comments Has local vocabulary available

#6
Name date
Label Date
URI http://purl.org/dc/terms/date
Definition Date that the DMP was completed.
Obligation & Occurrence Required
Example <date>20180910</date>
Comments Standardized; follows ISO 8601 date format

#7
Name analysisUnit
Label Subject
URI http://www.ddialliance.org/Specification/DDI-CV/AnalysisUnit_1.0.0_Genericode1.0_DDI-CVProfile1.0.xml
Definition Subject of the research described within the DMP.
Obligation & Occurrence Required, Repeatable
Example <analysisUnit termsource="LCSH" termsourceid="sh85035858">Data processing service centers</analysisUnit>
Comments Has LCSH attributes and local vocabulary available

#8
Name storage
Label Storage
URI http://www.utkmetadata.org/specification/utk_namespace/storage
Definition All locations where the data will be stored (physical and digital storage).
Obligation & Occurrence Required, Repeatable
Example <storage>Reynholm Industries Data Repository</storage>
Comments No controlled vocabulary at this time

#9
Name description
Label Description
URI http://purl.org/dc/terms/description
Definition A short description of the nature of the project (does not need to be a full abstract).
Obligation & Occurrence Optional
Example <description>"This project entails a quantitative data fusion of aggregate date from data processing service centers owned by Reynholm Industries."</description>
Comments No controlled vocabulary; string input by user

#10
Name domain
Label Domain
URI http://purl.org/dc/terms/subject
Definition The domain area(s) of the research described in the DMP.
Obligation & Occurrence Required, Repeatable
Example <domain termsource="LCSH" termsourceid="sh89003285">Computer science</domain>
Comments Has LCSH attributes available

#11
Name softwarePackage
Label Analysis
URI http://www.ddialliance.org/Specification/DDI-CV/SoftwarePackage_1.0_Genericode1.0_DDI-CVProfile1.0.xml
Definition Software platforms for data analysis used in the research described in the DMP.
Obligation & Occurrence Required, Repeatable
Example <softwarePackage>SPSS</softwarePackage>
Comments Limited attributes from LCSH; local controlled vocabulary (see here)

#12
Name analysisType
Label Technique
URI http://www.utkmetadata.org/specification/utk_namespace/technique
Definition The specific data analysis technique(s) used to process the data.
Obligation & Occurrence Required, Repeatable
Example <analysisType termsource="LCSH" termsourceid="sh85012506">Bayesian statistical decision theory</analysisType>
Comments Has LCSH attributes and local vocabulary available

#13
Name embargoes
Label Embargoes
URI http://www.utkmetadata.org/specification/utk_namespace/embargoes
Definition Any applicable embargoes limiting data access (data from the project described in the DMP) for a period of time.
Obligation & Occurrence Optional
Example <embargoes>"The public will have access to the raw data one year after the project funding window closes."</embargoes>
Comments No controlled vocabulary; string input by user

#14
Name affiliation
Label Affiliation
URI http://www.utkmetadata.org/specification/utk_namespace/affiliation
Definition Departmental affiliations of the researchers with refinements for associated organizations.
Obligation & Occurrence Required, Repeatable
Example <affiliation>Department of Computer Science, University College London</affiliation>
Comments LCNAF attributes and local vocabulary available

#15
Name dataType
Label Type
URI http://www.ddialliance.org/Specification/DDI-CV/DataType_1.0_Genericode1.0_DDI-CVProfile1.0.xml
Definition The types of data included in the research.
Obligation & Occurrence Required, Repeatable
Example <dataType termsource="LCSH" termsourceid="sh2007000909">Quantitative</dataType>
Comments DDI and LCSH attributes available

#16
Name byteSize
Label Size
URI http://www.w3.org/TR/vocab-dcat-2/#Property:distribution_size
Definition The total expected size of the data produced in the research, in bytes.
Obligation & Occurrence Required, Repeatable
Example <byteSize>4 TB</byteSize>
Comments DCAT attributes and local controlled vocabulary available

#17
Name accessRights
Label Reuse
URI http://purl.org/dc/terms/accessRights
Definition Guidelines for reusing the DMP, including attribution and modification.
Obligation & Occurrence Required
Example <accessRights>"This DMP may be reused with attribution to creator Matt Berry."</accessRights>
Comments No controlled vocabulary; string input by user

#18
Name rights
Label Rights
URI http://purl.org/dc/terms/rights
Definition Definition & Guidelines for data reuse outlined within the DMP.
Obligation & Occurrence Required
Example <rights>"The public will have access to aggregate data tables and intermediate processed data."</rights>
Comments No controlled vocabulary; string input by user

#19
Name staffWrap
Label Staff Wrapper
URI http://www.utkmetadata.org/wrapper/utk_namespace/staffWrap
Definition Wrapper element containing PI, creator, contributor, affiliation.
Obligation & Occurrence The wrapper element itself is required and not repeatable. The elements within the wrapper have their own obligations and occurences (see relevant specification tables above).
Example <staffWrap><PI>Allard, Suzie</PI><creator>Forrester,Amy</creator><contributor>Cannon, Leah</contributor><affiliation>UTK</affiliation></staffWrap>
Comments Attributes available for child elements - see relevant tables above

#20
Name analysisWrap
Label Analysis Wrapper
URI http://www.utkmetadata.org/wrapper/utk_namespace/analysisWrap
Definition Wrapper element containing softwarePackage and analysisType.
Obligation & Occurrence The wrapper element itself is required and not repeatable. The elements within the wrapper have their own obligations and occurences (see relevant specification tables above).
Example <analysisWrap><softwarePackage>SPSS</softwarePackage><analysisType>Bayesian statistical decision theory</analysisType></analysisWrap>
Comments Attributes available for child elements - see relevant tables above

#21
Name rightsReuseWrap
Label Rights and Reuse Wrapper
URI http://www.utkmetadata.org/wrapper/utk_namespace/rightsReuseWrap
Definition Wrapper element containing accessRights and rights.
Obligation & Occurrence The wrapper element itself is required and not repeatable. The elements within the wrapper have their own obligations and occurences (see relevant specification tables above).
Example <rightsReuseWrap><accessRights>"This DMP may be reused with attribution to creator Matt Berry."</accessRights><rights>"The public will have access to aggregate data tables and intermediate processed data."</rights></rightsReuseWrap>
Comments Attributes available for child elements - see relevant tables above
